## 一、游戏类型
以「强化学习智能体养成」为核心的**实时策略挂机游戏**。玩家通过自定义奖励函数塑造智能体的行为逻辑，挂机完成模型训练后，可组建多智能体阵容与其他玩家进行实时非回合制对战。
核心特色：低门槛RL玩法（新手可视化调参）+ 高自由度自定义（进阶API开放）+ 挂机养成的轻量化体验，差异化主打「AI策略进化」的乐趣内核。

目标平台：支持CUDA的PC

## 二、核心玩法设计
### 2.1 基础玩法循环
玩家选择官方预训练模型 → 配置奖励函数与训练参数 → 后台挂机持续训练智能体 → 冻结训练完成的模型组建对战阵容 → 在线匹配与其他玩家的智能体对战(客户端推理并上传模型决策,服务端进行战斗并返回结果) → 根据对战结果迭代优化奖励设计。

### 2.2 多智能体训练机制
- 支持多个智能体在同一环境中联合训练，每个智能体可独立设置奖励函数，同步完成策略迭代。
- 支持冻结已训练完成的智能体并放入训练环境，作为陪练、队友或固定对手，实现自博弈式的策略迭代，提升智能体的泛化能力。
- 训练全程支持可视化观察，可自由切换视角查看任意智能体的动作、状态与实时奖励。
- 支持多个风格模型蒸馏融合为一个综合模型

### 2.3 技能系统
玩家可放玩家专属的技能增强参与感，技能可影响己方和敌方智能体。在训练环境中技能可通过配置带有随机参数的规则(作为环境的一部分进行域随机化)来自动释放。
智能体本身可配置固定个数的多种智能体技能

### 2.4 对战规则
- 除了单个智能体对战外，还支持多智能体阵容对战，玩家可将多个已冻结的训练好的智能体组队上场。
- **单局设置硬性时间上限**，强制对局终止，从规则层面避免玩家设计「纯苟活」类无限拖延的取巧奖励，保证对战的对抗性。
- 胜负判定结合击杀数、剩余血量、目标占点等综合规则，与时间限制共同约束奖励设计的合理边界。

## 三、核心技术架构
### 3.1 算法选型：分层架构体系，兼顾落地性与泛化能力
#### 选型背景
游戏环境存在动态非平稳特性（对手策略多样、赛季机制更新、玩家自定义奖励），需同时满足**细粒度奖励兼容、强泛化能力、挂机训练效率、渲染训练分离**四大核心诉求。

##### 方案1：PPO + 浅层GRU + 域随机化
主流方案：
1. **奖励容错性高**：PPO 标准实现中内置优势归一化（Advantage Normalization），将每批次优势值标准化至零均值单位方差，使策略更新对奖励绝对尺度不敏感，玩家自定义奖励即使权重差距一个数量级也不易导致训练崩溃，大幅降低新手调参门槛。配合近端策略裁剪（Policy Clipping）限制单步更新幅度，进一步保障训练稳定性。
2. **泛化能力优异**：搭配1~2层GRU时序网络与分层域随机化，智能体具备短期记忆能力，可通过数帧交互快速适配陌生对手风格，应对动态环境的表现远超纯MLP架构的确定性算法。
3. **训练平滑稳定**：策略提升呈线性渐进趋势，无长时间平台期，契合挂机养成的成长反馈需求，玩家挂机后可直观看到能力指标稳步提升。

##### SAC-Discrete（最大熵+集成Critic+域随机化）
保留SAC原生优势，针对泛化与奖励鲁棒性做定向增强：
1. **原生核心优势**：
   - Off-policy特性完美适配挂机场景：可复用历史经验数据，样本效率高于on-policy算法，玩家挂机积累的历史数据可持续用于模型迭代。
   - 最大熵正则化适配细粒度自定义奖励：鼓励智能体探索多样化策略，降低玩家奖励设计不合理导致的局部最优问题，提升奖励微调容错性与对战策略多样性。
2. **增强改造点**：
   - 新增自动奖励归一化，解决原生SAC对奖励尺度敏感的问题，适配玩家自定义场景；
   - 采用集成Critic网络，取最小值作为价值估计，降低过拟合风险，提升策略鲁棒性；
   - 叠加分层域随机化与历史模型自博弈池，大幅提升环境泛化能力，避免过拟合单一陪练风格。

#### 3.1.1 奖励修改后的经验回放缓冲区处理方案
如果是PPO，直接丢弃旧缓冲区
如果是SAC，旧域数据在重新计算`r'`后依然能复用，持续强化通用动力学认知
> 注意对于SAC重新计算后的旧数据应作为新数据, 旧数据不丢弃而是根据TD误差进行权重衰减并逐渐淘汰

#### 3.1.2 模型细节
- Dual-clip PPO：在PPO基础上增加第二个裁剪参数c（c=3），当优势函数为负且新旧策略比过大时，限制方差
- 多头价值估计（Multi-Head Value）：将奖励分解为5个类别（Farm相关、KDA相关、伤害相关、推塔相关、胜负相关），分别估计价值后加权求和
- 训练时价值函数使用全信息（包括不可见对手信息），推理时仅使用策略网络

> 多教师策略蒸馏（Multi-Teacher Policy Distillation），将多个教师模型的知识蒸馏到单一学生模型（学生驱动蒸馏）之后继续训练

### 3.2 技术栈选型
- **模型训练与推理**：采用 libtorch，与游戏引擎同属C++技术栈，无需依赖Python环境，可在游戏进程内一体化实现训练+推理，打包分发便捷，同时可选CUDA GPU加速。
- **游戏引擎**：采用 cocos2dx，与libtorch无缝集成。
- **模型结构**：采用「结构化低维观测向量+多层感知机（MLP）」的基础架构，不使用CNN处理游戏图像，大幅降低算力开销，适配端侧挂机训练的硬件条件。
  - 堆叠`8~16`帧观测向量作为输入，搭配`1~2`层GRU时序网络，兼顾时序记忆能力与推理效率

### 3.3 核心技术解决方案
#### 3.3.1 训练泛化方案
针对「对战环境动态变化、对手策略未知、赛季机制更新」的泛化需求，采用多层级泛化策略，优先低成本方案，逐步进阶：
1. **分层域随机化（Domain Randomization）**
   训练阶段每个episode随机扰动环境参数，强制策略学习环境通用规律，而非过拟合单一环境细节，实现成本最低、效果显著：
   - 随机出生位置、初始状态、对手风格（官方预训练多类型模型随机抽取），保障基础对战泛化性；
   - 技能数值浮动、地图障碍物随机布局，进一步提升策略鲁棒性，训练难度同步提升；
   - 新地图、新机制通过更新预训练底座承接，玩家基于新模型微调即可，无需从零重训。
2. **GRU短期记忆机制**
   使用浅层GRU替代纯MLP骨干，输入堆叠多帧观测序列，让智能体具备短期时序记忆能力：
   - 对战中可通过数帧交互快速推断陌生对手的行为风格并调整策略，具备few-shot适应能力
3. **策略泛化：历史模型自博弈池**
   系统自动保存玩家训练过程中的10个历史版本模型，训练时随机抽取作为对手/队友加入环境：
   - 避免智能体过拟合当前陪练的单一风格，实现自博弈式的持续进化，提升策略多样性与泛化能力。

> Self-Trajectory Augmentation可作为辅助泛化手段，通过对历史轨迹添加观测噪声、动作扰动生成增强样本，提升策略鲁棒性，实现成本低，可与上述方案叠加使用。



#### 3.3.2 可视化方案
沿用「训练-渲染分离」的核心架构：

渲染端部署独立的纯推理模型实例，与训练端解耦；训练端全速运行采样训练，每5~10个episode向渲染端同步一次最新模型参数，保障可视化效果与训练进度基本同步。
   - 支持参数硬拷贝
   - 推理实例独立维护GRU隐藏状态，不影响训练端的采样流程。

#### 3.3.3 多智能体训练方案
1. **独立智能体**
   每个智能体维护独立的模型与专属奖励函数，将其他智能体视为环境动态的一部分：
   - 完美契合「每个智能体独立配置奖励」的核心玩法，实现逻辑简单，训练稳定性强；
   - 支持任意数量智能体同环境联合训练，适配不同阵容规模的训练需求。
2. **中心化训练**
   针对同阵营合作场景，提供可选的中心化训练模式（类MASAC/MAPPO架构）：
   - 训练阶段Critic网络接入全阵营的全局观测信息，Actor网络仍仅使用自身观测输出动作；
   - 训练出的智能体团队配合能力显著提升，适合组队团战、占点防守等合作类场景，作为进阶功能向玩家开放。
3. **冻结模型机制**
   训练完成的智能体可冻结网络参数（关闭梯度计算），仅作为前向推理实体加入训练环境：
   - 支持充当队友、对手、陪练NPC等多种角色，实现自博弈迭代训练；
   - 冻结模型兼容所有训练模式，可灵活搭配自定义训练场景。

#### 3.3.4 多环境并行与消融对比
1. **多环境并行训练**
   支持同时启动多个独立的游戏环境并行采样，所有环境的样本统一汇入训练缓冲区：
   - 大幅提升采样效率，加速模型收敛，充分利用端侧CPU/GPU算力；
   - 并行环境数量支持玩家自定义配置，适配不同硬件性能的设备。
2. **消融对比功能**
   支持创建多个独立训练实例，每个实例可配置不同的奖励参数、超参数、模型结构，并行训练：
   - UI支持分屏同步渲染对比，直观展示不同配置下智能体的行为差异；
   - 训练结束后自动生成对比报告，量化展示场均击杀、存活时长、技能命中率等指标的差异，帮助玩家验证参数效果，降低调参门槛。

> 挂机程序不应该占用太多资源，需要由用户进行配置

## 四、核心系统设计
### 4.1 统一接口规范
设定标准化的**观测向量接口**与**动作空间接口**，作为所有智能体模型的统一契约：
- **观测向量**：固定维度与语义，包含自身状态、敌方相对状态、全局环境状态三类字段；
- **动作空间**：固定维度，包含移动控制、技能释放的离散动作。移动使用WASD型离散动作, 与人类操作习惯一致。
统一接口是整个项目的架构基石，保证预训练模型通用、玩家自定义模型可无缝接入对战、奖励修改不影响模型结构。

### 4.2 预训练模型系统
提供多种不同行为倾向的官方预训练模型（进攻型、防守型、辅助型、均衡型等），玩家可直接选用作为训练起点，跳过随机探索的冷启动阶段，快速获得「修改奖励→行为变化」的正向反馈。
所有预训练模型基于统一接口规范训练，兼容所有奖励配置与对战场景。

### 4.3 奖励配置系统
采用「分层设计，兼顾新手与进阶玩家」的配置体系：
1. **预设奖励模板**：提供官方设计的多套成熟奖励模板（击杀导向、存活导向、辅助导向、运营导向等）
2. **可视化滑块配置**：将奖励拆解为「造成伤害、受到伤害、存活时长、距离敌方、技能命中、击杀目标、对局分数」等基础分项以及基于规则的自定义项，玩家通过滑块/蓝图调整各分项权重，无需代码基础即可自定义奖励逻辑。
3. **API开放接口**：针对进阶玩家，开放接口，支持自定义复杂奖励函数、修改模型超参数、导入导出模型文件，保留高自由度与社区生态空间。
4. **奖励归一化** GAE（λ=0.95）
  系统内置奖励归一化逻辑，对玩家配置的奖励函数做自适应处理，保障训练稳定性，可关闭：
   - 自动统计最近N步的奖励均值与方差，将奖励缩放至固定合理区间，避免奖励尺度过大/过小导致训练崩溃；
   - 设置单步奖励上下限，拦截极端奖励配置，防止策略退化
